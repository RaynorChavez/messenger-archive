# Messenger Archive - Project Plan

A self-hosted system to archive Facebook Messenger group chats in real-time, with a web interface for browsing, searching, and (eventually) AI-powered organization.

## Overview

This project bridges a Facebook Messenger group chat to a Matrix homeserver using mautrix-meta, then archives all messages to PostgreSQL with a Python service. A Next.js SPA provides a searchable, browsable interface.

    timestamp               TIMESTAMP NOT NULL,

-- Indexes for common queries
CREATE INDEX idx_messages_timestamp ON messages(timestamp DESC);
CREATE INDEX idx_messages_sender ON messages(sender_id);
CREATE INDEX idx_messages_room ON messages(room_id);
CREATE INDEX idx_messages_reply ON messages(reply_to_message_id);
CREATE INDEX idx_messages_content_search ON messages USING gin(to_tsvector('english', content));

## Authentication Flow

### Messenger Bridge Auth (mautrix-meta)

1. Bridge uses cookie-based authentication
2. User logs into Facebook in browser
3. Extracts session cookies via browser devtools (or mautrix-manager UI)
4. Provides cookies to bridge via Matrix bot chat
5. Bridge maintains persistent connection to Messenger

**Session behavior:**
- Sessions don't auto-expire
- May disconnect if: password reset, logout elsewhere, Meta flags suspicious activity
- Re-auth required occasionally (typically every few months)
- 2FA recommended to reduce Meta security flags

### SPA Auth

1. User visits any page
2. If no valid session cookie â†’ redirect to `/login`
3. User enters shared password
4. Backend verifies against bcrypt hash
5. Backend sets HTTP-only session cookie
6. User can access all pages

## API Endpoints

### Authentication

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/auth/login` | Login with password |
| POST | `/api/auth/logout` | Clear session |
| GET | `/api/auth/me` | Check auth status |
| POST | `/api/auth/change-password` | Update password |

### Messages

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/messages` | List messages (paginated, filterable) |
| GET | `/api/messages/:id` | Get single message |
| GET | `/api/messages/search` | Full-text search |

### Threads

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/threads` | List threads (auto-detected reply chains) |
| GET | `/api/threads/:id` | Get thread with all messages |

### People

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/people` | List people with stats |
| GET | `/api/people/:id` | Get person with recent messages |
| PATCH | `/api/people/:id` | Update person notes |

### Stats

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/stats` | Dashboard stats |
| GET | `/api/stats/activity` | 30-day activity data |

### Settings

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/settings/status` | Bridge & sync status |

## Deployment

### Oracle Cloud Free Tier

**Resources (Always Free):**
- ARM VM: Up to 4 OCPUs, 24GB RAM (we use 2 OCPU, 12GB)
- Block storage: 200GB
- Egress: 10TB/month

### Provisioning Steps

1. **Create Oracle Cloud account**
   - Sign up at cloud.oracle.com
   - Requires credit card (not charged for free tier)

2. **Create ARM VM instance**
   ```
   Shape: VM.Standard.A1.Flex
   OCPUs: 2
   Memory: 12 GB
   Image: Ubuntu 22.04 (aarch64)
   Storage: 100GB boot volume
   ```

3. **Configure networking**
   - Open ports: 22 (SSH), 80 (HTTP), 443 (HTTPS)
   - Assign public IP

4. **Install Docker & Docker Compose**
   ```bash
   sudo apt update && sudo apt upgrade -y
   sudo apt install docker.io docker-compose-v2 -y
   sudo usermod -aG docker $USER
   ```

5. **Clone repo & configure**
   ```bash
   git clone <repo-url> messenger-archive
   cd messenger-archive
   cp .env.example .env
   # Edit .env with your values
   ```

6. **Start services**
   ```bash
   docker compose -f docker-compose.prod.yml up -d
   ```

7. **Configure reverse proxy (Caddy/nginx)**
   - Point domain to VM IP
   - Auto-HTTPS with Let's Encrypt

8. **Authenticate Messenger bridge**
   - Access mautrix-manager or use Matrix client
   - Follow cookie extraction flow
   - Verify connection

## Security Considerations

1. **Password storage**: bcrypt with cost factor 12
2. **Session tokens**: HTTP-only, secure cookies
3. **Database**: Not exposed externally, internal Docker network
4. **HTTPS**: Enforced via Caddy in production
5. **Messenger cookies**: Stored encrypted in mautrix-meta database
6. **Rate limiting**: Applied on auth endpoints
